# センサデータ処理 — 前処理・特徴量設計・カラス検出ロジック

> **今回のスコープ: カラスの検出のみ。** 「人によるゴミ出し」など他状態の分類は将来拡張として記載する。処理はすべて FRDM-MCXN947 上で行う。

## サーマルデータ

入力：

```text
32×24 = 768点
```

前処理：

```text
32×24
  ↓ 縮小
16×12
  ↓ 背景差分
  ↓ 時間差分
  ↓ 特徴量化
```

特徴量候補：

- 平均温度変化量
- 最大温度変化量
- 変化領域の面積
- 熱源の重心位置
- 熱源の移動量
- 一定時間内の変化継続時間

---

## 距離データ

入力：

```text
8×8 = 64点
```

前処理：

```text
8×8距離マップ
  ↓ 背景差分
  ↓ 時間差分
  ↓ 特徴量化
```

特徴量候補：

- 平均距離変化量
- 最大距離変化量
- 変化領域の面積
- 変化領域の重心位置
- ゴミ袋表面の形状変化
- 小刻みな動きの有無

---

## カラス検出ロジック（今回のスコープ）

単一のセンサだけではなく、サーマルと距離の両方を組み合わせて判定する（センサフュージョン）。今回は「カラスによるゴミ漁り」を検出する／しないの2値判定に絞る。

### カラスによるゴミ漁り（検出対象）

```text
サーマル差分あり
+ 距離差分あり
+ 小刻みな変化が継続
+ 小〜中サイズの熱源
+ ゴミ袋周辺の部分的な距離変化

=> カラスによるゴミ漁りと判定
```

### 通常状態（非検出）

```text
背景との差分がほぼない
+ 時間変化が小さい

=> 通常状態（カラスなし）
```

### 将来拡張: 人によるゴミ出し

今回のスコープ外。学習データとロジックを追加して区別できるようにする予定。

```text
大きな熱源
+ 大きな距離変化
+ 短時間で変化が収束

=> 人によるゴミ出し（将来分類）
```

---

## 判定アプローチの段階的拡張

1. **ルールベース（まず確実に動かす）**
   背景差分・時間差分・重心・継続時間などのしきい値でカラスの有無を判定する。FRDM-MCXN947 上で実装し、特徴量の妥当性を確認する。

2. **エッジAI（eIQ / Neutron NPU）**
   収集した特徴量・差分マップを学習データとして、FRDM-MCXN947 の eIQ で軽量モデルを推論する。必要に応じて eIQ Neutron NPU を活用し、低消費電力でカラス検出を行う。サーマルと距離を統合判定する構成を想定。

---

## データ収集の方針（カラス検出向け）

- まず「通常状態」の背景データを十分に取得する
- ダミーカラス（ヒーター内蔵）を用いて「カラスゴミ漁り」状態のデータを取得する
- 各状態でサーマル差分マップ・距離差分マップ・抽出特徴量を記録する
- （将来）人がゴミ袋を操作する状態のデータも追加取得し、多状態分類へ拡張する
