# センサデータ処理 — 前処理・特徴量設計・検出ロジック

## サーマルデータ

入力：

```text
32×24 = 768点
```

前処理：

```text
32×24
  ↓ 縮小
16×12
  ↓ 背景差分
  ↓ 時間差分
  ↓ 特徴量化
```

特徴量候補：

- 平均温度変化量
- 最大温度変化量
- 変化領域の面積
- 熱源の重心位置
- 熱源の移動量
- 一定時間内の変化継続時間

---

## 距離データ

入力：

```text
8×8 = 64点
```

前処理：

```text
8×8距離マップ
  ↓ 背景差分
  ↓ 時間差分
  ↓ 特徴量化
```

特徴量候補：

- 平均距離変化量
- 最大距離変化量
- 変化領域の面積
- 変化領域の重心位置
- ゴミ袋表面の形状変化
- 小刻みな動きの有無

---

## 検出ロジック

TerraGuard AI は、単一のセンサだけではなく、サーマルと距離の両方を組み合わせて判定する（センサフュージョン）。

### カラスによるゴミ漁り

```text
サーマル差分あり
+ 距離差分あり
+ 小刻みな変化が継続
+ 小〜中サイズの熱源
+ ゴミ袋周辺の部分的な距離変化

=> カラスによるゴミ漁りの可能性が高い
```

### 人によるゴミ出し

```text
大きな熱源
+ 大きな距離変化
+ 短時間で変化が収束

=> 人によるゴミ出しの可能性
```

### 通常状態

```text
背景との差分がほぼない
+ 時間変化が小さい

=> 通常状態
```

---

## 判定アプローチの段階的拡張

1. **ルールベース（初期）**
   背景差分・時間差分・重心・継続時間などのしきい値で 2〜3 状態を分類する。プロトタイプ検証と特徴量の妥当性確認に用いる。

2. **エッジAI（本番 / FRDM-MCXN947）**
   収集した特徴量・差分マップを学習データとして、MCU 上で軽量モデルを推論する。サーマルモデルと距離モデルを統合判定する構成を想定。

---

## データ収集の方針

- まず「通常状態」の背景データを十分に取得する
- ダミーカラス（ヒーター内蔵）を用いて「カラスゴミ漁り」状態のデータを取得する
- 人がゴミ袋を操作する「人によるゴミ出し」状態のデータを取得する
- 各状態でサーマル差分マップ・距離差分マップ・抽出特徴量を記録する
