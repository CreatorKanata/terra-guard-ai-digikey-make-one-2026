## 1. システム全体図 — すべてが「1 チップ」に集約

```text
       ┌──────────────────────┐                ┌──────────────────────┐
       │ MLX90640 (サーマル)   │                │ VL53L5CX (ToF 距離)   │
       │ 32×24 温度アレイ      │                │ 8×8 マルチゾーン距離   │
       └──────────┬───────────┘                └──────────┬───────────┘
                  │ I2C (3.3V, 同一バス FC2/LPI2C2)        │
                  └──────────────────┬────────────────────┘
                                     ▼
        ╔════════════════════════════════════════════════════════════╗
        ║            NXP FRDM-MCXN947  （★この 1 枚で完結★）          ║
        ║  ┌──────────────────────────────────────────────────────┐  ║
        ║  │ デュアル Arm Cortex-M33 (最大150MHz) + DSP            │  ║
        ║  │ ① センサ取得  ② 前処理(回転/crop/背景差分/前景化)     │  ║
        ║  │ ③ 4ch テンソル整形 (24×24×4 int8)                    │  ║
        ║  ├──────────────────────────────────────────────────────┤  ║
        ║  │ eIQ Neutron NPU  ④ 小型CNN 推論 → [not_crow, crow]   │  ║
        ║  └──────────────────────────────────────────────────────┘  ║
        ║  ⑤ 時間デバウンス(約1.4秒連続でcrow確定)                     ║
        ║  ⑥ ステータスLED 点灯 / シリアル(INFER)出力                  ║
        ╚════════════════════════════════════════════════════════════╝
                                     │
                       (クラウド・外部ボード・カメラは一切なし)
```

- **クラウド 0 / 追加ボード 0 / カメラ 0**。センサは FRDM-MCXN947 に**直接 I2C 接続**。
- 推論は CPU ではなく**オンチップの eIQ Neutron NPU** で実行（低消費電力・低レイテンシ）。
- 出力は**ローカルで完結**（LED 点灯 ＋ デバッグ用シリアル）。画像はボード外に一切出ない。

## 2. ハードウェア構成

### 2-1. 中核ボード — NXP FRDM-MCXN947（おすすめ製品）

| 項目 | 内容 |
| --- | --- |
| MCU | MCXN947（**デュアル Arm Cortex-M33、最大150MHz** + DSP コプロセッサ） |
| AIアクセラレータ | **eIQ Neutron NPU**（エッジAI推論を高速・低消費電力で実行） |
| メモリ | 最大 2MB デュアルバンク Flash |
| センサ接続 | **I2C / I3C**（MLX90640・VL53L5CX を直接接続） |
| デバッガ | オンボード **MCU-Link**（USB 1 本で書き込み・デバッグ。追加プローブ不要） |
| 開発環境 | **MCUXpresso SDK**（CMake / armgcc、LinkServer 書き込み） |

**このボードを選んだ理由**: 「センサ直結」「エッジAI（NPU）」「単体で開発・デバッグ完結」が 1 枚で揃い、
**“1 チップでエッジAIプロダクトを成立させる”** という本作のコンセプトに最適だったため。
本コンテストのおすすめ製品でもあります。

### 2-2. センサ

| センサ | 役割 | 解像度 / IF | 検出での意味 |
| --- | --- | --- | --- |
| **MLX90640**（110°版） | サーマル（温度分布） | 32×24 / I2C(0x33) | カラスの**羽毛表面 約30〜35℃**を熱源として捉える |
| **VL53L5CX** | ToF 距離（奥行き分布） | 8×8 / I2C(0x29) | カラスが**背景より手前に出た**ことを距離差で捉える |

- 2 センサは**同一 I2C バス（FC2 / LPI2C2, P4_0=SDA / P4_1=SCL）**に共存（アドレス衝突なし）。
- いずれも **3.3V 直結 OK**（レベル変換不要）。Breakout 基板のプルアップを利用。
- **カメラを使わない**ことが本作の肝。32×24＋8×8 の低解像度では人物を判別できる映像が原理的に成立せず、**プライバシーが構造的に守られる**。

### 2-3. 配線（実機疎通済み）

外部センサは **J8 ヘッダ pin1〜4** に最短接続（追加ハンダ付け不要）。

```text
FRDM-MCXN947  J8           センサ (MLX90640 / VL53L5CX)
  pin1 (P3V3, 3.3V) ────── VCC
  pin2 (GND)        ────── GND
  pin3 (P4_1, SCL)  ────── SCL
  pin4 (P4_0, SDA)  ────── SDA
```

## 3. 検出パイプライン（1 フレームの流れ）

実装（`src/FRDM-MCXN947/terra-guard-ai/app/`）に忠実なデータフローです。

```text
① 取得         サーマル32×24[℃]            距離8×8[mm]
                  │                            │
② 整形      90度右回転＋中央24行crop      左右反転＋無効ホールド
            → 24×24=576画素[℃]            → 8×8=64ゾーン[mm]
                  │                            │
③ 背景差分   thermal_fg=max(0,cur-bg)     dist_fg=max(0,bg-cur)
            (背景より暖かい正側)          (背景より手前の正側)
                  └──────────┬─────────────────┘
                             ▼
④ 4ch整形    24×24×4 int8 テンソル
              ch0 thermal_abs (絶対温度)
              ch1 thermal_fg  (サーマル前景)
              ch2 distance     (距離8×8→最近傍3倍で24×24)
              ch3 distance_fg  (距離前景8×8→24×24)
                             ▼
⑤ NPU推論    eIQ Neutron NPU 小型CNN → Softmax [not_crow, crow]
                             ▼
⑥ デバウンス  約1.4秒（≒10フレーム）連続 crow で「確定crow」
                             ▼
⑦ 出力        ステータスLED点灯 ＋ INFER 行をシリアル出力
              （= 次工程「緑色レーザー追い払い」へのトリガー）
```

- 完成サーマルフレームは実測 **約7fps**。NPU 推論はサーマル完成ごと（約7回/秒）に駆動。
- **なぜ温度と距離を 1 つの AI に束ねるか**: カラスは「暖かくて・かつ・背景より手前に出た・小さなまとまり」。
  単一センサでは日なたの壁（温度のみ）や人・物の通過（距離のみ）と区別できない。
  **4ch を 1 つの CNN に同時入力**して、その同時条件を学習で取り出すのが核心。

## 4. AIモデル（eIQ Neutron NPU 上の小型CNN）

```text
Input [24,24,4]
  Conv 8  3x3 relu → MaxPool2     → 12x12x8
  Conv 16 3x3 relu → MaxPool2     → 6x6x16
  Conv 24 3x3 relu → AvgPool(6,6) → Flatten → 24
  Dense 2 softmax
Output [1,2] = [p_not_crow, p_crow]
```

- **2クラス**（not_crow / crow）の小型 CNN。**full-integer int8 量子化**して NPU 向けに変換。
- NPU 非対応 op を避ける設計（GlobalAvgPool → 明示サイズ AvgPool で代替 等）。
- **ファームの前処理と学習側の前処理を 1 対 1 で厳密一致**させ、実機判定が学習時とずれないようにしている。

## 5. 開発手法 — “1 回目”の完全AIファームウェア開発

本作は、コードの実装からビルド・実機への書き込み・AIモデルの学習まで、
**開発のすべての工程を AI コーディングエージェント Claude Code がハンドリング**しました。
**人間（制作者）は一切手を動かしていません**——コードを 1 行も書かず・1 度も開かず、
ビルドも・書き込みも・学習コマンドの実行もしていません。

| 工程 | 担当 | 人間の関与 |
| --- | --- | --- |
| コード実装 | C / MCUXpresso SDK を **AI が記述** | 0 行（1 度も閲覧せず） |
| ビルド | `cmake --build`（CMake/armgcc）を **AI が CLI 実行** | なし |
| 書き込み（flash） | LinkServer による実機書き込みを **AI が CLI 実行** | なし |
| 学習コマンド | データ収集・int8量子化学習・NPU変換・ヘッダ生成を **AI が CLI 実行** | なし |
| 実機確認 | シリアルログ（921600）を **AI が取得・解析**して切り分け | USB 抜き差し等の物理操作のみ |

これを支えた仕掛け:

1. **開発プロジェクトの土台**: FRDM-MCXN947 のビルド・書き込み環境（MCUXpresso SDK プロジェクト一式）を整備し、AI が手を動かせる足場を用意。
2. **FRDM-MCXN947 開発用スキルの自作**: ビルド／書き込み／シリアル／I2C 配線／NPU 学習の**確定手順とハマりどころ**を AI 向けスキルに集約。AI はスキルを起動するだけで毎回ブレずに実機開発できる。
