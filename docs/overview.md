# 概要 — 課題・ソリューション・全体アーキテクチャ

## 解決したい課題

都市部や住宅地では、カラスがゴミ袋を破り、ゴミ置き場を荒らす被害が発生している。従来の対策はネット、物理カバー、人による監視が中心であり、カラスの学習能力や設置環境によって効果が限定的になることがある。

カメラAIによる監視も考えられるが、住宅地では通行人や住民の映像を取得してしまうため、プライバシー面の課題がある。また、クラウドAIは通信環境、消費電力、運用コストの問題もある。

TerraGuard AI は、低解像度センサとオンデバイスAIを用いることで、カメラを使わず、クラウドにも依存しない害鳥・害獣検知を目指す。

---

## ソリューション概要

TerraGuard AI は、ゴミ置き場などの監視対象エリアに設置し、以下の情報を取得する。

- サーマルセンサによる温度分布
- ToF距離センサによる奥行き分布
- 背景との差分
- 時間方向の変化
- 変化領域の重心位置や移動量

本システムは、カラスをカメラ画像で直接認識するのではなく、ゴミ置き場に発生する状態変化を学習する。

分類対象の例：

```text
1. 通常状態
2. カラスによるゴミ漁り
3. 人によるゴミ出し
```

**今回のスコープでは「カラスによるゴミ漁り」の検出に絞って確実に動作させる。** その後、追加学習により「人によるゴミ出し」を含む多状態分類へ拡張する。

## 全体アーキテクチャ

DigiKey Make ONE Challenge 2026 版では、**NXP FRDM-MCXN947 単体**で、センサを直接接続してセンサフュージョンとエッジAI処理を行う。外部のArduino等の中継ボードは使用しない。

FRDM-MCXN947 は **eIQ Neutron NPU** を内蔵しており、低消費電力でエッジAI推論を実行できる。

```text
  MLX90640 (サーマル 32×24)
        │ I2C
        ▼
  ┌─────────────────────────────────────┐
  │  NXP FRDM-MCXN947                    │
  │   - Cortex-M33 ×2 + DSP             │
  │   - eIQ Neutron NPU (エッジAI推論)   │
  │                                     │
  │   センサ取得 → 前処理(差分/特徴量)     │
  │     → カラス検出AI/ルール判定         │
  └─────────────────────────────────────┘
        ▲                    │
        │ I2C                │ 検出結果
        │                    ▼
  VL53L5CX (ToF 8×8)   追い払い機構（後続: LED/ブザー/サーボ等）
```

> 今回は「カラス検出」まで。追い払い機構の駆動は後続ステップ（[roadmap.md](./roadmap.md)）。

## 重点ポイント（DigiKey Make ONE Challenge 2026）

- NXP MCU 単体でのセンサフュージョン（サーマル + 距離）
- eIQ Neutron NPU を活かしたエッジAI推論
- 害鳥・害獣対策ソリューションとしての実用性
- DigiKey で入手可能な部品構成
- 検知から追い払いまでの一連の動作デモ（最終形）
- カメラを使わないプライバシー配慮型検知
